📝 深度笔记：FreeRTOS 任务调度机制
1. 宏观理解：它是如何“同时”运行多个任务的？
FreeRTOS 是一个抢占式实时操作系统。它的调度核心在于：高优先级的任务一旦就绪，必须立刻剥夺低优先级任务的 CPU 使用权。

SysTick 中断：操作系统的“心跳”，每隔固定时间（通常 1ms）触发一次，检查是否有更高优先级的任务需要运行。

PendSV 中断：真正的“搬运工”，负责保存当前任务的上下文（寄存器状态），并切换到下一个任务。

2. 关键概念：任务状态转换
在学习调度时，必须理清这四种状态的切换逻辑：

Running (运行态)：正在占用 CPU。

Ready (就绪态)：万事俱备，只欠 CPU（优先级不够）。

Blocked (阻塞态)：在等待某个事件（如 vTaskDelay 定时到期或等待信号量）。

Suspended (挂起态)：彻底“冬眠”，除非手动唤醒。
